# 千幕输出格式

前端根据这些字段展示审片、任务、角色世界和写入文本。周期、期限与时机字段从当前剧情语境中自然提炼，呈现未来可能的发展走向。

`progress` 为**本幕进度**（当前叙事单元的完成度，0-100）。

## 模块职能（去同质化，互不渗透）

世界不围绕{{user}}运转，但{{user}}保有最大参与自由。每个模块各锁唯一职能与视角距离：

- `quests`（任务）：{{user}} 此刻能伸手做什么。第一人称、向心视角，是 {{user}} 主动触碰世界的入口。
- `npc_updates`（角色动向）：某个具体的人此刻在过自己的日子。离心视角、聚焦个体能动。半数以上 NPC 与 {{user}} 暂无交集。
- `world_updates`（世界回声）：系统/集体层面的宏观位移（势力、天候、经济、公共事件）。最远全景视角，无主角。
- `chain_reactions`（因果链）：把各模块的事顺成可见连锁（A 触发 B、B 波及 C）。全局唯一可写"蝴蝶效应"处，多数链条在世界内部流转、与 {{user}} 无关。
- `director_comment`（众声）：每次随机化身一位带个性的看客（作家/导演/戏中NPC/磕学家/读者/毒舌影评人等），以其口吻议论本幕，开头点明身份，避免助手腔。

数量硬性下限：任务≥5、角色动向≥5、世界回声≥3、因果链≥3。

## 暗线注入

推演成功后，插件会把本结果提炼为紧凑的「暗线灵感池」，经由 `setExtensionPrompt`（或拦截器兜底）注入后续生成上下文，注入文案明确声明"不强制发生、不要求全部触发、不围绕任何人运转"，供主模型在自然时机抽取任意一点。可在幕后界面逐项开关并调整注入深度。

```json
{
  "story_status": {
    "title": "当前故事标题",
    "current_arc": "当前主线篇章",
    "current_stage": "当前阶段与下一步可能走向",
    "cycle": "贴合剧情语境的时间跨度或节奏名",
    "progress": 0,
    "mood": "一句话氛围",
    "summary": "勾人的楔子式引子（非流水账复述），点出当下暗涌与未定走向"
  },
  "quests": [],
  "npc_updates": [],
  "world_updates": [],
  "chain_reactions": [],
  "director_comment": "众声：随机化身一位带个性的看客以其口吻议论本幕，开头点明身份"
}
```

可选活幕字段（对应开关开启时才追加）：`threads`（伏笔显影）、`world_chatter`（尘寰群生）。

## 幕外 · 番外小剧场

独立于推演的加演模块，与千幕主功能完全隔离：**不写入聊天、不参与 `setExtensionPrompt` 注入、不进入推演提示词**。仅借用千幕的 API（可选独立的剧场 API 预设）与一个关联预设（勾选条目作为文风滤镜 + 设定来源），依据用户自由撰写的「此幕指令」生成一段番外正文。正文支持纯文字与可交互 HTML，由独立的沙箱阅读浮层（`iframe sandbox`）呈现；可保存到剧札、收藏。日志并入主日志列表，以「小剧场」标签与「推演」区分。
