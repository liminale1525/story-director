# 千幕输出格式

前端根据这些字段展示审片、任务节点、角色世界和写入文本。周期、期限与时机字段从当前剧情语境中自然提炼，呈现未来可能的发展走向。

审片的四个圆形评价维度**固定**为专业影视评判标准：**张力、情感、悬念、节奏**，模型只给出 0-100 数值；`progress` 为**本幕进度**（当前叙事单元的完成度）。

## 视角分工（去中心化平衡）

世界不围绕{{user}}运转，但{{user}}保有最大参与自由：

- `quests.inject_prompt`：以{{user}}第一人称视角书写——这是{{user}}主动伸手触碰世界的入口。
- `story_nodes.inject_prompt`：在第一人称与场景化全知视角间自然切换，事件可与{{user}}直接相关、间接波及或发生在视野边缘。
- `npc_updates / world_updates` 的 `inject_prompt` 与 `impact`：全知导演镜头，描述事件如何在世界中自行展开发酵；{{user}}可在场、旁观、间接受影响或毫不知情，事件不等待任何人。
- 蝴蝶效应：条目之间允许互相牵动，一个任务的发展可以改写某条世界事件的走向，小事件允许引发连锁变化。

## 暗线注入

推演成功后，插件会把本结果提炼为紧凑的「暗线灵感池」，经由 `setExtensionPrompt`（或拦截器兜底）注入后续生成上下文，注入文案明确声明"不强制发生、不要求全部触发、不围绕任何人运转"，供主模型在自然时机抽取任意一点。可在幕后界面开关并调整注入深度。


## 幕外 · 番外小剧场

独立于推演的加演模块，与千幕主功能完全隔离：**不写入聊天、不参与 `setExtensionPrompt` 注入、不进入推演提示词**。仅借用千幕的 API（可选独立的剧场 API 预设）与一个关联预设（勾选条目作为文风滤镜 + 设定来源），依据用户自由撰写的「此幕指令」生成一段番外正文。正文支持纯文字与可交互 HTML，由独立的沙箱阅读浮层（`iframe sandbox`）呈现；可保存到剧札、收藏。日志并入主日志列表，以「小剧场」标签与「推演」区分。
